
Network Topology Documentation

1. Overview
This topology represents a structured enterprise LAN segmented into departments — Servers, Marketing, and Engineering — all interconnected via a core switch. The network also integrates physical network access, an ArubaCX switch (management), and a wireless access point.

2. Core Components

2.1 Core Switch (Cisco vIOS)
- IP Address: 192.168.1.60
- Function: Central Layer 3 switch connecting all departmental switches, the Netprobe server, and the physical network.
- Interfaces:
  - Gi0/1: Physical Network / EVENG-SERVER (192.168.1.250)
  - Gi0/2: Servers Switch (192.168.1.66)
  - Gi0/3: Marketing Switch (192.168.1.67)
  - Gi0/0: Engineering Switch (192.168.1.68)
  - Gi1/0: Netprobe port e0
  - Gi1/1: Netprobe port e1

3. Departmental Switches

3.1 Servers Switch (Cisco vIOS)
- IP Address: 192.168.1.66
- Connected Devices:
  - Gi0/1 → (SRV-DC) – eth0
  - Gi0/2 → (SRV-APP) – eth0
  - Gi0/3 → (SRV-FILE) – eth0

3.2 Marketing Switch (Cisco vIOS)
- IP Address: 192.168.1.67
- Connected Devices:
  - Gi0/1 → PC-MKTNG – eth0
  - Gi0/2 → PRNT-MKTNG – eth0

3.3 Engineering Switch (Cisco vIOS)
- IP Address: 192.168.1.68
- Connected Devices:
  - Gi0/1 → ENG-WKSTN – eth0
  - Gi0/2 → Access Point (192.168.1.122) – Gi0/0
  - Gi0/3 → ArubaCX Switch IP: 192.168.1.143 - (Mgmt port)

4. Additional Devices

4.1 EVENG-SERVER
- IP Address: 192.168.1.250
- Purpose: Runs the network emulation platform and connects to the core switch through the physical network.

4.2 Netprobe
- IP Address: 192.168.1.50
- Purpose: Network monitoring or testing server connected to the core switch.

4.3 ArubaCX Switch (Aruba CX)
- IP Address: 192.168.1.143
- Connection: Mgmt port connected to Engineering switch Gi0/3.
- Function: Managed switch for network operations/testing.

4.4 Access Point (vIOS L3)
- IP Address: 192.168.1.122
- Function: Provides wireless access for Engineering or other departments.

6. How Services Connect (EVE-NG Lab Environment)

IMPORTANT: This is an EVE-NG virtual lab with a flat Layer 2 network design.

6.1 Core Network Flow:
- Core Switch (192.168.1.60) → Central aggregation point (Layer 2 switching only)
- Three access switches connect to Core Switch for basic switching
- Default gateway: 192.168.1.3 (EVE-NG management network)
- All devices in single broadcast domain: 192.168.1.0/24

6.2 Access Switch Connections:

Servers Switch (- 192.168.1.66):
- Function: Basic Layer 2 access switching for virtual servers
- Configuration: SNMP communities (public/private), syslog to 192.168.1.100
- Connected services:
  * SRV-DC → Virtual domain controller/server
  * SRV-APP → Virtual application server  
  * SRV-FILE → Virtual file server

Marketing Switch (192.168.1.67):
- Function: Basic Layer 2 access switching for marketing department
- Configuration: SNMP communities (public/private), syslog to 192.168.1.100
- Connected services:
  * PC-MKTNG → Virtual marketing workstation
  * PRNT-MKTNG → Virtual network printer

Engineering Switch (AccSW-Eng - 192.168.1.68):
- Function: Basic Layer 2 access switching for engineering department
- Configuration: SNMP communities (public/private), syslog to 192.168.1.100
- Connected services:
  * ENG-WKSTN → Virtual engineering workstation
  * Access Point (192.168.1.122) → Simulated wireless AP with basic IP/SNMP config
  * ArubaCX Switch (192.168.1.143) → Dummy test device for practicing network operations

6.3 Monitoring and Testing Services:

Netprobe (192.168.1.50):
- Function: Network monitoring and connectivity testing tool
- Connection: Dual-homed to Core Switch (Gi1/0 and Gi1/1)
- Purpose: Used for network diagnostics and monitoring

EVE-NG Server (192.168.1.250):
- Function: Physical host running the virtual lab environment
- Connection: Connected via Core Switch to lab management network
- Purpose: Lab platform providing virtualization

Monitoring Server (192.168.1.100):
- Function: Centralized logging destination
- Purpose: Receives syslog messages from all switches

6.4 Service Architecture Summary:
- Network Type: Flat Layer 2 switching (no VLANs or routing)
- Switch Function: Basic access switching with management IPs
- Virtual Devices: All PCs/servers are EVE-NG virtual nodes for testing
- Test Targets: ArubaCX switch serves as practice device for network operations
- Monitoring: SNMP and syslog configured on all managed devices
